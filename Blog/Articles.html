Z-Value

October 4, 12:00 PM | Ethan Blomgren

When I was working on the layout, I had problems with pushing divs forward and backwards as they overlapped. I had applied opacity to the article tag and the sticky attribute to the header. After asking for help in the e-mail, I learned that it has to do with a quirky property called z-index. When a select few properties like opacity are altered, it changes the z-index of the element. The z-index can be overridden using the z-index value; integers only. Z-index is only accounted for when an element's position value has been set to something non-static. One thing I could not work out was getting the padding and margin around the h1 tag to shrink as the header went from static to sticky.

Google Fonts

October 2 | Ethan Blomgren

Google has free, professional looking fonts you can load into your CSS from their online font library. Absolute purists consider using nonstandard display fonts to be a source of CSS web bloat, so their use should be restrained to a healthy amount. These fonts can be loaded into an html document by linking to them in the head as stylesheets and referring to them using the font family parameters.

Images

October 7 | Ethan Blomgren

Images are relatively simple to manage in HTML. They can either be referenced to within the file system of the website or loaded in from a link on the web. I decided to pick a background image from my computer, a still from the movie Porco Rosso, and upload it to a free image hosting site. Using links only works with the raw link to the file itself, as in, a URL ending with an image file extension. There are a host of properties in CSS which can alter the way images can be rendered to the screen. WEBP is a great lossy image format for web because the compression rate is high while retaining decent fidelity, but it is not supported in Safari on Mac devices older than Big Sur because of feuding between Apple and Google.
